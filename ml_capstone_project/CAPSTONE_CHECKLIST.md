## 1️⃣ 技术能力（40分）

### 1.1 数据诊断能力（10分）

#### ✅ 核心检查项

- [ ] **数据规模诊断**：能够快速判断数据量级（小/中/大样本），并据此调整策略
  - 小样本(<1000)：避免复杂模型，考虑数据增强
  - 中样本(1K-100K)：常规方法适用
  - 大样本(>100K)：可使用复杂模型，考虑采样策略

- [ ] **数据质量检查**：能系统性检查数据完整性和准确性
  - 缺失值：比例、分布、缺失模式（MCAR/MAR/MNAR）
  - 重复值：识别并处理完全重复/部分重复样本
  - 异常值：使用3σ、IQR、LOF等方法检测异常

- [ ] **特征类型识别**：准确区分数值、分类、时间、文本特征
  - 数值特征：连续 vs 离散，分布类型（正态/偏态/多峰）
  - 分类特征：有序 vs 无序，基数（低/中/高）
  - 时间特征：时间戳、日期、时间间隔
  - 文本特征：短文本 vs 长文本，结构化 vs 非结构化

- [ ] **分布分析**：理解特征分布特性并采取相应处理
  - 偏态分布：是否需要对数变换/Box-Cox变换
  - 长尾分布：是否需要截断/分箱
  - 多峰分布：是否暗示潜在子群体

#### 📝 自我检验方法

**任务**：给定新数据集，在30分钟内完成数据诊断报告，包含：
1. 数据规模、特征类型统计
2. 数据质量问题汇总（缺失/重复/异常）
3. 关键特征的分布可视化
4. 初步的数据问题解决方案建议

**评分标准**：
- 10分：完成所有检查项，诊断准确，可视化清晰，建议合理
- 7-9分：大部分检查项完成，少量遗漏
- 4-6分：基本检查完成，但深度不足
- 0-3分：仅完成表面检查，缺少系统性分析

### 1.2 特征工程能力（10分）

#### ✅ 核心检查项

- [ ] **数值特征处理**：掌握标准化、归一化、分箱等技术
  - 标准化(StandardScaler)：适用于正态分布特征
  - 归一化(MinMaxScaler)：适用于有明确边界的特征
  - 鲁棒缩放(RobustScaler)：适用于有异常值的数据
  - 分箱(Binning)：将连续变量离散化

- [ ] **分类特征编码**：选择合适的编码方式
  - One-Hot编码：低基数(<10)无序分类
  - Label编码：有序分类特征
  - Target编码：高基数(>10)分类特征
  - Frequency编码：基于频率的编码

- [ ] **特征交叉与衍生**：创造有业务意义的新特征
  - 交叉特征：组合两个或多个特征（如年龄×收入）
  - 聚合特征：分组统计（如用户历史平均消费）
  - 时间特征：提取年/月/日/星期/节假日等
  - 文本特征：TF-IDF、词嵌入、主题模型

- [ ] **特征选择**：降低维度，避免过拟合
  - 过滤法：相关系数、卡方检验、互信息
  - 包裹法：递归特征消除(RFE)
  - 嵌入法：L1正则化、树模型特征重要性

#### 📝 自我检验方法

**任务**：对给定数据集进行特征工程，要求：
1. 处理所有原始特征（数值+分类）
2. 创造至少5个有意义的衍生特征
3. 进行特征选择，给出Top 10重要特征
4. 使用处理后的特征训练模型，相比原始特征提升性能≥5%

**评分标准**：
- 10分：所有特征处理正确，衍生特征有创意且有效，模型提升显著
- 7-9分：特征处理基本正确，有少量衍生特征，模型有提升
- 4-6分：完成基础处理，但缺少衍生特征或特征选择
- 0-3分：仅完成简单的编码，无特征创造

**得分**：____ / 10

### 1.3 算法选择能力（10分）

#### ✅ 核心检查项

- [ ] **问题类型判断**：准确识别ML任务类型
  - 监督学习：分类（二分类/多分类）、回归
  - 无监督学习：聚类、降维、异常检测
  - 混合方法：先聚类再分类、异常特征+分类

- [ ] **算法适用性评估**：根据数据特点选择算法
  - **线性 vs 非线性**：特征与目标是否线性可分
  - **样本量**：小样本避免深度模型，大样本可用复杂模型
  - **特征维度**：高维数据考虑降维或L1正则化
  - **类别平衡**：不平衡数据需要重采样或算法调整

- [ ] **算法特性理解**：理解每个算法的优缺点
  - **可解释性**：线性模型 > 树模型 > 神经网络
  - **训练速度**：朴素贝叶斯/KNN快 > 树模型 > SVM慢
  - **预测速度**：树模型快 > 线性模型 > KNN慢
  - **鲁棒性**：集成方法 > 单一模型

- [ ] **Baseline建立**：先用简单模型建立基线
  - 分类任务：Logistic Regression、Decision Tree
  - 回归任务：Linear Regression、Ridge
  - 聚类任务：K-Means
  - 用简单模型快速验证特征有效性

#### 📝 自我检验方法

**任务**：给定5个不同的数据集（包含分类、回归、聚类任务），要求：
1. 正确判断每个任务的类型
2. 为每个任务选择3个候选算法，并说明理由
3. 实现并对比这些算法的性能
4. 给出最终推荐算法及选择依据

**评分标准**：
- 10分：任务类型判断全对，算法选择合理且有深入分析，性能对比充分
- 7-9分：大部分判断正确，算法选择基本合理
- 4-6分：基本判断正确，但算法选择缺乏系统性
- 0-3分：判断错误较多，算法选择随意

### 1.4 模型评估能力（10分）

#### ✅ 核心检查项

- [ ] **评估指标选择**：根据业务目标选择合适指标
  - **分类任务**：
    - 平衡数据：Accuracy
    - 不平衡数据：Precision/Recall/F1/AUC
    - 多分类：Macro/Micro/Weighted平均
  - **回归任务**：
    - MAE：对异常值鲁棒
    - MSE/RMSE：惩罚大误差
    - R²：解释方差比例
  - **聚类任务**：
    - 轮廓系数(Silhouette Score)
    - Calinski-Harabasz指数
    - Davies-Bouldin指数

- [ ] **交叉验证**：避免过拟合，获得稳定评估
  - K折交叉验证：常规方法
  - 分层交叉验证：类别不平衡时使用
  - 时间序列交叉验证：时序数据专用
  - 留一法(LOO)：小样本数据

- [ ] **模型诊断**：识别过拟合/欠拟合
  - 学习曲线：观察训练集/验证集性能随样本量变化
  - 验证曲线：观察性能随超参数变化
  - 混淆矩阵：分析具体错误类型
  - 残差分析：回归模型的误差分布

- [ ] **超参数调优**：系统性地优化模型
  - 网格搜索(GridSearchCV)：小参数空间
  - 随机搜索(RandomizedSearchCV)：大参数空间
  - 贝叶斯优化(Optuna/Hyperopt)：高效搜索
  - 理解关键参数的影响（如树深度、正则化强度）

#### 📝 自我检验方法

**任务**：对一个分类/回归任务进行完整的模型评估，要求：
1. 选择至少3个评估指标，并解释为何选择这些指标
2. 使用5折交叉验证评估模型稳定性
3. 绘制学习曲线和验证曲线，分析模型状态
4. 进行超参数调优，并展示调优前后的性能对比
5. 给出模型的最终评估报告（包括各项指标、可视化、结论）

**评分标准**：
- 10分：指标选择合理，评估过程完整，诊断深入，调优有效
- 7-9分：评估基本完整，有部分诊断分析
- 4-6分：完成基础评估，但缺少深入诊断或调优
- 0-3分：仅计算基本指标，无系统性评估

## 2️⃣ 决策能力（30分）

### 2.1 数据问题诊断决策（10分）

#### ✅ 核心检查项

- [ ] **遇到新数据时，能快速建立诊断流程**
  - 步骤1：数据规模和类型检查
  - 步骤2：数据质量问题识别
  - 步骤3：分布和关系分析
  - 步骤4：问题优先级排序

- [ ] **针对不同数据问题，能选择合适的解决方案**
  - **缺失值**：
    - MCAR → 删除/均值填充
    - MAR → 模型预测填充
    - MNAR → 建模为特殊类别
  - **异常值**：
    - 数据错误 → 删除
    - 真实极端值 → 保留或稳健处理
  - **类别不平衡**：
    - 轻度(1:3) → 调整类别权重
    - 中度(1:10) → SMOTE过采样
    - 重度(1:100) → 异常检测方法

- [ ] **能权衡不同处理策略的利弊**
  - 删除 vs 填充：样本量 vs 信息损失
  - 简单填充 vs 模型填充：计算成本 vs 准确性
  - 过采样 vs 欠采样 vs 算法调整

**情景模拟**：给定3个有不同数据问题的数据集，要求：

3. 分析每种方案的优缺点
4. 选择最佳方案并实施，对比处理前后的效果

**评分标准**：

- 4-6分：能识别问题，但方案单一或缺乏对比

## 4️⃣ 业务能力（10分）

### 4.1 技术指标到业务价值的转化（10分）

#### ✅ 核心检查项

- [ ] **能将技术指标转化为业务语言**
  - AUC=0.85 → "模型能正确识别85%的潜在客户"
  - Recall=0.90 → "90%的欺诈交易能被检测出"
  - MAE=500 → "预测销售额的平均误差为500元"

- [ ] **能评估模型的业务价值**
  - 计算成本收益：
    - 假阳性成本（误判带来的损失）
    - 假阴性成本（漏判带来的损失）
    - 模型部署和维护成本
  - 与baseline对比（如人工审核、规则系统）
  - ROI分析：投入 vs 收益

- [ ] **能提出可落地的建议**
  - 不仅给出模型，还给出使用建议
  - 如：阈值设置（Precision vs Recall权衡）
  - 如：哪些场景下使用模型，哪些场景人工介入
  - 如：模型监控和更新策略

- [ ] **能识别业务约束并调整方案**
  - 实时性要求 → 选择快速算法
