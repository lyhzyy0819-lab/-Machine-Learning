## 💻 代码规范与注释标准

### 1. 注释密度要求

**原则：宁可注释过多，也不能让学习者看不懂**

- ✅ **每个函数/类**必须有完整的 docstring 说明
- ✅ **核心算法步骤**必须逐行或逐块注释
- ✅ **数学公式**对应的代码实现必须标注公式编号或说明
- ✅ **参数含义**要在定义处详细解释
- ✅ **关键变量**的物理意义要注释清楚

### 2. 注释示例标准

#### ❌ 不符合要求的注释（过于简单）
```python
# 计算距离
distances = np.sqrt(np.sum((X - point)**2, axis=1))
```

#### ✅ 符合要求的注释（详细教学）
```python
# 计算每个训练样本到预测点的欧氏距离
# 公式: d = sqrt((x1-x2)^2 + (y1-y2)^2 + ... + (xn-yn)^2)
#
# 步骤分解:
# 1. (X - point): 训练集每个点与预测点的差值，形状 (n_samples, n_features)
# 2. **2: 将差值平方
# 3. np.sum(..., axis=1): 对每个样本的所有特征维度求和，得到 (n_samples,)
# 4. np.sqrt: 开平方根得到最终距离
distances = np.sqrt(np.sum((X - point)**2, axis=1))
```

### 3. 代码组织结构

每个 Notebook 应包含：

1. **理论讲解** - 算法原理、数学推导、直观理解
2. **从零实现** - 用 NumPy 手写核心算法（加深理解）
3. **Sklearn 实现** - 使用标准库的生产级实现
4. **可视化** - 算法过程可视化、结果对比图表
5. **真实案例** - 在真实数据集上的应用
6. **练习题** - 巩固学习的实践任务

### 4. 变量命名规范

- 使用**有意义的英文名称**（不用拼音）
- 关键变量在定义时**添加类型和形状注释**

```python
# 训练数据特征矩阵, shape: (n_samples, n_features)
X_train = ...

# 训练数据标签向量, shape: (n_samples,)
y_train = ...

# 学习率 (learning rate)
learning_rate = 0.01

# 迭代次数 (number of iterations)
n_iterations = 1000
```

## 🤖 AI 助手协助指导原则

当使用 Claude 或其他 AI 助手为本项目编写/补全代码时，请遵循以下原则：

### 1. 教学为先

- **不要**直接给出最优化、最简洁的代码
- **应该**给出易于理解、步骤清晰的代码
- **优先**展示算法的底层逻辑，而非高度封装的调用

### 2. 详细注释要求

- 每个代码块必须包含：
  - **目的说明**：这段代码在做什么？
  - **步骤拆解**：关键步骤的逐行解释
  - **参数解释**：输入输出的形状、含义
  - **数学对应**：与理论公式的对应关系

### 3. 从零实现优先

- 核心算法**必须先用 NumPy/纯 Python 实现**
- 实现后再对比 sklearn 等库的版本
- 让学习者理解"黑盒"背后的原理

### 4. 可视化丰富

- 算法过程的**动态演示**或分步可视化
- 参数变化对结果的**影响对比图**
- 数据分布、决策边界的**直观展示**

### 5. 真实数据集

- 优先使用 sklearn.datasets 或公开数据集
- 避免过度简化的玩具数据
- 体现算法在实际问题中的表现

### 6. 错误示范与纠正

- 适当展示**常见错误**（如不标准化数据、错误的参数选择）
- 对比**正确做法**，解释为什么
- 帮助学习者避坑

### 7. 中文优先

- 所有注释、说明、Markdown 文本使用**中文**
- 变量名、函数名使用**英文**（符合编程规范）
- 专业术语保留英文并附中文解释

### 给学习者

1. **按顺序学习**：从 `supervised_learning` 开始，再到 `unsupervised_learning`
